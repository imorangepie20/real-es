# 매물 거래 일정 재구성 설계 (일정 + 대금 통합)

**작성:** 2026-07-07
**선행:** [2026-06-24-property-management-design.md](2026-06-24-property-management-design.md) (매물 관리·fields.ts 단일 소스), [2026-06-25-contract-workflow-design.md](2026-06-25-contract-workflow-design.md) (계약 양식·체크리스트)

## 1. 목표

매물 폼의 "일정" 그룹을 실거래 흐름(계약 → 중도금 → 잔금 → 입주)에 맞게 재구성하고, 계약서 수준의 **대금 스케줄**(계약금·중도금·잔금 = 금액+일자 쌍)과 임대차 **만기일**을 추가한다. 거래유형(매매/임대차)에 따라 무관한 필드는 숨긴다.

## 2. 배경 — 현재 문제

`fields.ts` "일정" 그룹: 계약희망일 → 계약일 → **입주희망일 → 입주일** → 중도금1 → 중도금2 → 잔금일.

1. **시간 순서 어긋남**: 입주가 중도금·잔금보다 앞. (캘린더 파생 `property-events.ts`는 이미 올바른 순서)
2. **희망일(협의)과 확정일(마일스톤)이 뒤섞임.**
3. **거래유형 무시**: 중도금은 매매 전용인데 전월세에도 노출, 전월세 핵심인 만기일은 부재.
4. **금액 부재**: 중도금1·2는 날짜만 있고 금액이 없음. 계약서 인쇄의 계약금·잔금은 수기 기입란.

## 3. 결정 — 평면 컬럼 확장 (A안)

별도 대금 테이블(N차 중도금·지급 추적) 대신 **Property 평면 컬럼 확장**을 택한다.

- 근거: `fields.ts` 평면 키 단일 소스가 폼·그리드·엑셀·계약서를 구동 — 평면 컬럼이면 전부 자동 연동. 중도금은 실무 대부분 0~2회. 지급 추적은 기존 계약진행 체크리스트와 역할 중복.
- 트레이드오프: 중도금 최대 2회 고정.

## 4. 스키마 (비파괴 마이그레이션 `add_payment_schedule`)

```prisma
downPayment    BigInt?  // 계약금(원)
interim1Amount BigInt?  // 중도금1(원)
interim2Amount BigInt?  // 중도금2(원)
balanceAmount  BigInt?  // 잔금(원)
leaseEndDate   String?  // 임대차 만기일(YYYYMMDD)
```

기존 날짜 필드 7개(contractHopeDate·contractDate·moveInHopeDate·moveInDate·interim1Date·interim2Date·balanceDate)는 이름·데이터 그대로. 금액은 기존 money 규약(만원 입력·표시, 원 저장) 따름.

## 5. 폼 "일정" 그룹 재구성 (fields.ts)

lg 기준 한 행 4칸(span 3), 실거래 흐름 순서:

| 행 | 필드 (라벨) |
|---|---|
| 협의·계약 | 계약희망일 · 입주희망일 · 계약일 · **계약금** |
| 중도금 *(매매만)* | **중도금1**(금액) · 중도금1일 · **중도금2**(금액) · 중도금2일 |
| 잔금·입주 | **잔금**(금액) · 잔금일 · 입주일 · 만기일 *(임대차만)* |

- 배열 순서 = 위 표 순서. 금액+일자 쌍이 인접, 시간순 유지.
- 라벨 변경: `interim1Date` "중도금1"→"중도금1일", `interim2Date` "중도금2"→"중도금2일" (금액 필드가 "중도금1·2" 라벨을 가져감).
- **거래유형 조건 노출**: `PropertyField`에 `trades?: string[]` 추가(해당 거래유형에서만 폼 노출, 미지정=항상). 중도금 4필드 `["A1"]`, 만기일 `["B1","B2","B3"]`. 거래유형 미선택 시 전부 표시. 그리드·엑셀 매칭은 trades 무시(전 필드 유지).
- 겸사 정리: 보증금 라벨 전환 `RENTAL`이 B1·B2만 인식 → B3(단기임대) 포함.

## 6. 파생 연동

- **캘린더**(`property-events.ts` + `calendar/actions.ts` select): `leaseEndDate`(만기일) 추가 — 입주일 뒤. 갱신 영업 타이밍용 실무 핵심.
- **계약서 인쇄**(`contract-forms.ts` + `contract-document.tsx`): 매매계약서 계약금(수기)→`downPayment`, 잔금(수기)→`balanceAmount`, 중도금 금액→`interim1Amount`(+2). 임대차계약서 계약금·잔금 동일. 저장값 있으면 자동기입, 없으면 기존처럼 수기란.
- **그리드·엑셀**: fields.ts에서 자동 파생. ⚠️ 엑셀 헤더 매칭이 라벨 기준 — 기존 엑셀의 "중도금1" 컬럼은 이제 금액으로 해석됨(날짜는 "중도금1일"). 문서화로 안내.

## 7. 합계 안내 (차단 없음)

매매(A1)에서 **계약금과 잔금이 둘 다 입력된 경우**, 입력된 대금 합계(계약금 + 입력된 중도금1·2 + 잔금)가 가격과 다르면 일정 그룹 하단에 안내 문구 표시(예: "대금 합계 8억 — 가격 8.5억과 다릅니다"). 저장은 차단하지 않는다. 중도금 미입력은 정상 케이스(계약금+잔금 거래).

## 8. 검증

1. `prisma migrate dev`(서버=라이브 즉시 반영) → 스키마 확인
2. `pnpm build` 통과
3. Playwright E2E: 매물 폼에서 매매 선택 → 중도금 노출·만기일 숨김 / 전세 선택 → 반대 확인, 값 저장·재편집 왕복
4. 계약서 인쇄 미리보기에서 계약금·잔금 자동기입 확인
