# 네이버 매물 검색 페이지 설계 (2026-06-24)

메뉴상 **"네이버 부동산 > 매물 검색"**(`/dashboard/naver`) 페이지를 제품 스펙([../../project_structure.md](../../project_structure.md) §네이버 매물 데이터 수집)에 맞춰 완성한다. 현재는 기능만 배선된 최소 버전 → 지도/표/엑셀/상태를 스펙 수준으로 끌어올린다.

## 0. 디자인 원칙
CLAUDE.md §6대로 **템플릿(SDTPL_ADM) 컴포넌트·패턴을 적극 재사용**한다 — 프리미티브 자작 금지, 간격·타이포·색·radius는 템플릿 토큰을 따른다. frontend-design 기준으로 **완성도 높게**: 표는 템플릿 `data-table` 패턴, 레이아웃은 기존 분할/툴바 패턴을 베이스로 삼고, 임의 px·hex 하드코딩 금지.

## 1. 범위
- **포함**: 검색 바, 좌우 분할 레이아웃, 지도(단지/클러스터 분기), 매물 표(검색·필터·정렬), 엑셀 출력 필드 선택, 상태 UX.
- **비범위(YAGNI)**: 검색 저장 목록 / 관심 매물 / 매물 관리·고객·일정 등 다른 메뉴. 데이터 정합성 하드닝(페이징 잘림 H1 등)은 별도 작업.

## 2. 흐름 (§데이터 수집)
`거래유형 → 매물유형 → 지역(시도→시군구→읍면동) → 추출 → 지도 + 표`

## 3. 레이아웃
```
┌─ 매물 검색 ────────────────────────────────────────────────┐
│ [거래: ●매매 ○전세 ○월세 ○단기]  [매물유형 ▾]              │  검색 바
│ [시/도 ▾][시군구 ▾][읍면동 ▾]                  [검색]      │
├──────────────────────────┬─────────────────────────────────┤
│  카카오 지도              │  결과 패널                       │
│  단지형: 단지 마커        │  단지형: 단지목록→선택→매물표    │
│  비단지형: 클러스터 숫자  │  비단지형: 매물표 바로           │
│  (지도 ↔ 표 클릭 연동)    │  [검색▢][필터][정렬]  [엑셀▾]    │
└──────────────────────────┴─────────────────────────────────┘
```
- 와이드: 좌(지도) + 우(결과) 2분할. 좁은 화면: 세로 스택.
- 상단에 현재 선택 요약 칩(거래·유형·동) + 결과 수.

## 4. 지도 분기 (핵심 — 현재 미구현)
- **단지형**(`propertyMode==='complex'`, A01/A02/A04): `complex/boundedComplexes`의 단지들을 **단지 마커**로 표시 → 마커/목록 클릭 → `complex/article/list` 매물 표.
- **비단지형**(`'article'`, C/D/E): `article/map/articleClusters`로 **클러스터(원 안 숫자 = 매물 수 합)** 표시 → 원 클릭 → `article/clusteredArticles` 하위 매물 표.
- 좌표 박스 = 동 중심 ±0.3° + `filter.legalDivisionNumbers:[naverCode]` (기존 `box.ts` 재사용).

## 5. 신규 스크레이퍼 (src/lib/naver, 기존 fetch/parse/cache 패턴)
- `getArticleClusters(naverCode, { realEstateTypes, tradeTypes })` → `{ clusters: [{ lat, lng, count, article? }], totalCount }`
- `getClusteredArticles(clusterId, filter, paging)` → `NaverArticle[]` (단일 매물 클러스터의 하위 목록)
- 브라우저 동일 헤더·페이싱·캐시·페이징 캡 규칙 동일 적용.

## 6. 매물 표 (§데이터 출력)
- `@tanstack/react-table`(이미 설치) 기반. 컬럼: 매물유형·거래·단지명·가격(억/만원)·전용/공급면적·층·동·중개사·확인일.
- 헤더 텍스트 검색, 컬럼 정렬, 거래/유형 필터. 가격은 억·만원 포맷.

## 7. 엑셀 (§엑셀출력)
- 다운로드 전 **출력 필드 선택 다이얼로그** → 선택 컬럼만 `export` route에 전달.

## 8. 상태 UX
- 로딩 스켈레톤, `0건 / 미수집 / 에러` 구분, 에러는 sonner 토스트, 선택 요약 칩.

## 9. 컴포넌트 분해
- `search-bar.tsx` — 거래·유형·지역·검색
- `map-panel.tsx` — 단지/클러스터 분기 + kakao
- `results-panel.tsx` — 단지목록 or 매물표 컨테이너
- `articles-table.tsx` — 필터/정렬(TanStack)
- `export-dialog.tsx` — 출력 필드 선택
- `collection-view.tsx` — 상태 오케스트레이션 + 레이아웃

## 10. 빌드 순서 (각 단계 독립 커밋 + 문서 갱신 §5)
1. **레이아웃** — 검색 바 + 좌우 분할 + 선택 요약/상태 UX (스크레이퍼 변경 없음)
2. **지도** — 단지 전체 마커 + 비단지형 클러스터 (신규 스크레이퍼 2종)
3. **표** — 검색·필터·정렬 (TanStack)
4. **엑셀** — 출력 필드 선택 다이얼로그
